# Support

This repository is maintained as a public AI automation and portfolio project.

## Getting Help

Before opening an issue:

1. read the README;
2. review the product overview;
3. check the setup notes;
4. test with demo users only;
5. confirm that no private parent or child data is included.

## What to Include in an Issue

- short problem description;
- steps to reproduce;
- expected behavior;
- actual behavior;
- whether the issue is related to Telegram, n8n, PostgreSQL, OpenAI, payments, or scheduling;
- safe logs with private data removed.

## What Not to Include

Do not include Telegram tokens, OpenAI keys, payment credentials, chat IDs, real user IDs, private parent messages, child names, webhook URLs, database passwords, or `.env` files.

## Production Note

A production deployment should include privacy controls, unsubscribe flow, request limits, payment validation, monitoring, and backups.

## Project Status

This is not a commercial support channel. Issues and improvements are handled as time allows.
